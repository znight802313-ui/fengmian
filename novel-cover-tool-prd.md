# 内部提效工具：小说封面生成器产品需求文档 (PRD)

## 1. 项目概述

本项目旨在为公司内部团队打造一款高效、美观的小说封面生成工具。通过整合大语言模型（生成高质量生图指令词）与专业生图模型（基于指令生成最终封面），帮助运营和作者快速产出符合网文业务审美标准的小说封面，极大降低设计门槛，提升包装效率。

## 2. 业务需求

### 2.1 目标用户与使用场景
- **目标用户**：公司内部运营人员、编辑、签约作者。
- **使用场景**：在小说入库、测试、上架或日常运营包装阶段，当缺少现成美术资源时，快速批量为书籍生成具有高级感、符合小说题材的封面图。

### 2.2 核心痛点与解决
- **痛点**：传统 AI 生图门槛高（需要自己写复杂的 Prompt）、比例和画风不稳定、单次出图试错成本高。
- **解决**：提供结构化表单（书名、简介、标签等）让 AI 自动写 Prompt；支持模型选择、尺寸比例切换、一次并发多张图片供挑选，提供即插即用的顺畅体验。

## 3. 技术栈推荐与部署方案

考虑到本项目为内部提效工具，且明确需要在腾讯云轻量服务器上线，推荐以下轻量级前端直连方案：

- **核心框架**：`Vite` + `React` (或 Vue3，视团队习惯而定，推荐 React 生态丰富)。
- **UI 框架**：`Tailwind CSS` + 搭配无头组件库（如 `Radix UI` 或 `shadcn/ui`），能够快速搭建出具有现代高级感的界面。
- **网络请求**：`Axios` 或原生 `fetch`，用于与各类大模型 API 直接交互。
- **部署方案**：
  - 前端打包为静态资源 (`npm run build`)。
  - 部署到**腾讯云轻量应用服务器**的 Nginx 目录下，或者使用腾讯云 Web 应用托管。
  - **安全性说明**：由于是内部工具，可将模型 API 密钥配置在前端项目的环境变量 (`.env`) 中并在构建时注入，使用者无需感知。如后续扩展，可加入简单的内网访问白名单或统一访问密码。

## 4. UI/UX 视觉设计要求

- **整体调性**：页面需要美观、高级、质感强。告别简陋的内部系统样式，打造“消费级”工具体验。
- **色彩规范**：采用**暖色温馨**的视觉设计。背景可使用带有细微暖光渐变的浅色调（如米白、浅暖灰、淡香槟色），按钮和重点元素使用琥珀色、暖橙或大地色系，营造舒适、放松的创作氛围。
- **布局设计**：推荐左右分栏（或经典的中轴线布局）：
  - **左侧/控制区**：小说信息输入、模型参数配置区，卡片式收纳，表单精致。
  - **右侧/预览区**：宽大的深色或柔和背景画板区，用于展示生成的图片矩阵，凸显图片质感。
- **交互细节**：加入微动画（Hover 阴影浮起、生成时的骨架屏或流光加载效果、图片加载淡入）。

## 5. 产品功能模块与交互流程

### 5.1 输入辅助区（小说信息）
用户可以手动输入或粘贴已有信息，用于辅助 AI 生成精准的画面描述：
- **书名（必填）**：影响画面主体、提示词以及兜底文字。
- **书籍标签/风格**：如“都市”、“玄幻”、“甜宠”等，影响画风。
- **简介信息**：提供给大语言模型提取视觉元素。

### 5.2 核心配置区
- **生图指令词模型选择器**：
  - 作用：将输入的小说信息转化为生图 Prompt。
  - 选项：接入现有各模型（Claude/Gemini/DeepSeek 等），**默认选中 `[次]gemini-3-pro-preview-thinking`**。
- **图片生成模型选择器**：
  - 作用：执行最终生图。
  - 选项：保留即梦 4.5、Gemini 4K 等，**默认选中 `[A]gemini-3-pro-image-preview-2k` (Gemini Image 2K)**。
- **尺寸比例选择**：
  - 选项：1:1（正方形）、3:4（竖图）、4:3（横图）、16:9（宽屏）、9:16（长竖屏）。
  - **默认选中：3:4（经典小说封面）**。
- **构图模板选择**：
  - 选项：角色特写、中景氛围感、宏大场景。
  - **默认选中：中景氛围感**。
- **生成张数选择**：
  - 选项：支持 1 到 4 张的步进器或单选。
  - **默认选中：2 张**。

### 5.3 交互主流程
1. **输入与配置**：用户填入书名、简介等信息，并按需调整右侧/下方的模型、尺寸、张数配置。
2. **两阶段生成**：
   - **阶段一（自动）**：用户点击“生成封面”按钮后，系统静默或弹框调用**生图指令词模型**，根据输入信息生成一段对应的视觉 Prompt。
   - **提示词编辑（可选）**：UI 应提供一个折叠面板或弹窗，允许进阶用户看到并修改这段 AI 生成的 Prompt。
   - **阶段二（并发）**：系统获取到最终 Prompt 后，根据用户选择的【生成张数】（如 2 张），**并发向生图模型发起 2 次请求**。
3. **展示与结果处理**：
   - 生成中：展示多个骨架屏或 Loading 占位图。
   - 生成后：呈现在画板中。用户可以点击某一张查看大图，并提供“下载”按钮（自动命名为 `[书名]_封面_1.png`）。

## 6. AI 生成策略与配置落地

### 6.1 生图指令词生成策略
- **输入组合**：系统提示词 (角色设定+风格指南) + 用户提示词 (书名+标签+简介+构图模板+目标尺寸)。
- **尺寸兼容**：原 PRD 中固定了 "3:4" 描述。新逻辑需动态替换，例如当用户选 16:9 时，提示词需变更为：“横版构图，16:9 比例，横向场景布局”。
- **API 接口**：对接配置好的代理网关（如 `https://once.novai.su/v1`），调用所选大模型。

### 6.2 并发生图与异常处理
- **并发策略**：使用 `Promise.all` 发起多张图片请求。针对 Gemini 接口，需观察并发 4 张是否触发限流（Rate Limit）。如遇频繁 429 报错，可在前端做简单队列控制或加入 1-2 秒的随机延迟 (Jitter) 错峰请求。
- **兜底重试**：Gemini 生图偶有解析不到图片的问题，代码需保留原有的 3 次失败重试机制。

## 7. 接口参数变更（对比老版本）

为了支持尺寸和动态配置，请求到 AI 模型的 Prompt 需要动态拼接参数：

```javascript
// 尺寸枚举映射
const ratioMap = {
  '1:1': '正方形构图，1:1比例',
  '3:4': '竖版构图，3:4比例，竖向书籍封面布局',
  '4:3': '横版构图，4:3比例',
  '16:9': '宽屏构图，16:9比例，影视感横向布局',
  '9:16': '长竖版构图，9:16比例，手机全屏布局'
};
```

## 8. 后续演进建议（v2.0）
1. **字体叠加引擎**：由于目前 AI 画中文易出错，后续可演进为“只生成无字底图”，前端引入 Canvas 提供字体选择（如站酷快乐体、毛笔行书），支持拖拽书名字体排版并导出。
2. **预设灵感库**：内置几十套高频小说的优秀 Prompt 模板，一键套用。
3. **历史记录保存**：引入浏览器的 IndexedDB 或服务器数据库，保存历史生成的满意图片。
